# 律师科技周刊

这里主要是阮一峰《科技爱好者周刊》的文章拆解与抽象。

## 整体框架结构（Macro Structure）

1.  **标题：** `律师科技周刊#期号：核心话题/主打文章标题`
2.  **卷首语（Intro / Metadata）：**
    *   作者信息（可以替换为你的名字：镐赞）
    *   发布时间/周期：记录每周值得分享的科技内容，周五早 8:00 企业微信推送（根据设定修改）
    *   简短声明/通知（如休息通知等）
    *   欢迎投稿及联系方式说明（邮箱/微信公众号）
3.  **封面图（Cover Image）：**
    *   一张精美的、有故事或科技感的配图
    *   图片说明及出处（via链接）
4.  **主打文章/深度长文（Hero Article / Deep Dive）：**
    *   占篇幅最大，深入探讨一个特定的科技与业务结合点、AI趋势或行业现象。
    *   具有较强逻辑推演，往往带有个人观点（例如：预测未来的 AI 编程、预测是新热点等）。
    *   小标题、分段结构、佐以引用或实例。
5.  **赞助商/软广插播（Sponsor / Ad Content - Optional）：**
    *   穿插在主文章后，介绍某款产品或服务（如：千问电脑版、国内大模型的跟进应用等）。
    *   图文并茂讲解使用方法及亮点，甚至提供试用建议。
6.  **科技动态（Tech News / Updates）：**
    *   简明扼要地列举 3-4 条本周有趣的科技圈时事或新奇发明。
    *   每条结构：小标题 + 1-2段简短描述 + （可选辅助图片）。
7.  **文章推荐（Articles）：**
    *   分享本周值得阅读的 5-8 篇文章（中英文皆有）。
    *   每条结构：数字序号 + `文章标题` + （中/英文标记） + 简短的推荐理由或摘要。
8.  **工具推荐（Tools）：**
    *   有趣的开源软件、网站、APP 等。
    *   每条结构：数字序号 + `工具名称` + 一两句话描述功能/特性 + （可选提交人 @xxx）。
9.  **AI 相关推荐（AI Tools - Optional, 依据栏目侧重）：**
    *   专门针对 AI 工具的一两个介绍。
10. **结尾声明 & References：**
    *   带（完）字样
    *   References (参考链接列表，按序号 [1][2] 排列，非常规范的 Markdown 链接提取)

## 针对律师受众的改造方案 (律所/法律人场景适配)

结合你（镐赞）的背景以及针对受众（律师、泛法律从业者），建议的《律师科技周刊》结构规范如下：

### 元数据集 / Markdown 前缀
```markdown
---
title: "律师科技周刊#期号：主打文章标题"
author: "镐赞"
date: "YYYY-MM-DD"
description: "这里记录每周值得分享的法律科技内容，周五早8:00发布。"
---
```

### 标准章节模块

- **1. 封面故事：**
  - `<一张有意义的图片>` + 一两句简炼解读（可与案卷、法庭、生活或科技美图相关）。
- **2. 卷首长文（本周深度）：**
  - **核心转变：** 将通用科技热点，**套入律师工作场景**。
  - 例如：本周 AI 新出了某个长文本处理模型，文章就探讨《如何用大模型帮你半小时完成数百页阅卷批注》。
  - 风格保持：少铺垫，多结论，直击痛点。
- **3. 行业动态（法律圈 x 科技）：**
  - 新出的司法科技工具、政策变化涉及的数据合规、某些大型律所的数字化动作。
- **4. 效率工具箱（工具推荐）：**
  - 为律师筛选的：PDF 提取器、录音转写增强版、庭审笔录辅助生成工具、文档排版工具等。
- **5. 值得一读（深度文章）：**
  - 挑几篇讲法律实务逻辑、或者关于自我提升/甚至法律科技前沿思考的好文。

## Skill: 律师科技周刊 生成器 规则抽象 (Skill Manifest Preparation)

我们要建立一个名为 `qingclaw-law-weekly-creator` 的技能。

### 角色设定
你是一个专业的法律科技内容主笔。你的任务是基于用户扔给你的散乱素材（网页链接、零碎想法、工具推荐、图片），将其整理、重写，并组装成符合《律师科技周刊》固定体裁的排版内容并返回。

### 输入格式要求
你需要接收用户提供的一组信息，至少包含：
- 期数及标题意向
- 长文素材或主题
- 科技/行业动态简讯（可多条）
- 工具/文章链接（可多个，带简要说明）

### 处理流程及输出规范
1. **自动提取与润色**：根据每个素材类别，用精炼、直白的中文改写。去除啰嗦部分。
2. **结构化拼装**：严格按照 `封面图` -> `深度长文` -> `行业动态` -> `工具推荐` -> `值得精读文章` 的层次输出。
3. **法律人视角改写**：如果用户扔来的是通用的 AI 工具，你需要在介绍时主动插入一两句 **“对律师的价值在哪”** 的点评（比如，“对审核合同特别有帮助”、“适合开庭前梳理时间线”等）。
4. **Markdown 格式**：全部输出采用优雅整洁的 Markdown（支持有序/无序列表、粗体高亮关键结论、大标题层级清晰）。
